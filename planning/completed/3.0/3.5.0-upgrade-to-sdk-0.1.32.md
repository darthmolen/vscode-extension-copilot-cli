# 3.5.0 — SDK 0.1.32 Upgrade

## Summary

This release integrates the SDK 0.1.32 changes that were already installed but not yet wired up. It adds explicit handlers for four new event types, fixes an early-event race condition during session creation, exposes `rpc.compaction.compact()` as a user-accessible `/compact` slash command, renders a `session.task_complete` indicator in the chat UI, and makes assistant reasoning break open tool groups (same as an assistant message already does).

## Phases

### Phase 1 — New SDK event handlers

Four event types currently fall through to the `default` log branch in `_handleSDKEvent()`:

| Event | Action |
|-------|--------|
| `subagent.deselected` | Log `[SDK Event]` prefix |
| `session.task_complete` | Log summary; fire `_onDidTaskComplete` emitter (Phase 4) |
| `session.background_tasks_changed` | Log `[SDK Event]` prefix |
| `system.notification` | Log `[SDK Event]` including `kind.description` |
| `permission.requested` | Log only |
| `permission.completed` | Log only |

### Phase 2 — `onEvent` race condition fix

Events fired during `createSession` (before the promise resolves) are currently dropped. The fix is to inject `onEvent: (event) => this._handleSDKEvent(event)` into the session config. The persistent `session.on()` listener registered in `setupSessionEventHandlers()` handles all subsequent events. Both can fire for the same event — `_handleSDKEvent` is idempotent.

### Phase 3 — `/compact` slash command

End-to-end implementation:
- `SDKSessionManager.compactSession()` — calls `session.rpc.compaction.compact()`
- VS Code command `copilot-cli-extension.compactSession`
- RPC message type `compact` in `shared/messages.ts`
- `onCompact` handler in `ExtensionRpcRouter.ts` + `chatViewProvider.ts`
- `CompactSlashHandlers.ts` (new service file)
- `CommandParser.js`: move `compact` from `not-supported` → `extension` event
- Help text updated in `InfoSlashHandlers.ts`

### Phase 4 — `session.task_complete` UI indicator

- New `_onDidTaskComplete` emitter + event on `SDKSessionManager`
- `TaskCompletePayload` in `shared/messages.ts`
- `sendTaskComplete()` on `ExtensionRpcRouter`
- Wired in `extension.ts` → webview via RPC
- `MessageDisplay.js` renders a distinct `task-complete` callout (icon + optional summary)

### Phase 5 — Reasoning breaks tool groups

One-line fix in `ToolExecution.js`: add `|| message.role === 'reasoning'` to the existing condition that closes the current tool group on user/assistant messages.

### Phase 6 — Version bump and testing

Bumped `package.json` to `3.5.0`. Full test suite: **1225 passing, 3 pending, 3 failing (all pre-existing baseline)**. Branch committed and pushed to `feature/3.5.0`.

### Phase 6a — Streaming spike (optional)

Exploratory spike to determine whether enabling `streaming: true` in `SessionConfig` produces `assistant.message_delta` events for normal sessions. Results gate Phase 6b.

## User Testing

1. /compact slash command

* Type /compact in the input — it should appear in the slash command panel (not show "not supported")
Select it and confirm it runs without error
* Check the Output Channel ("Copilot CLI") logs for [Compact] entries

2. Task-complete indicator

* After a session finishes a task (e.g., ask it to do something and let it complete), look for a distinct callout/badge in the message stream — a checkmark or "Task complete" label
* If the SDK fires session.task_complete, it should render inline in the chat

3. Reasoning breaks tool groups

* Trigger a response where the model reasons before or between tool calls
* The reasoning block should close the current tool group, starting a new one — rather than appearing inside an existing collapsed group

4. New SDK events (passive)

* Just run a normal session and scan the Output Channel for any [SDK Event] log lines — confirms subagent.deselected, system.notification, etc. are no longer logging "Unhandled event type"

5. Regression check

* Existing slash commands still work (/clear, /help, /plan, etc.)
* Tool calls still expand/collapse normally
* Session resume still works
